# 주 4일 근무 운영 관리 웹사이트 기획서

## 📋 프로젝트 개요

### 프로젝트명
주 4일 근무 및 휴무·반차 운영 관리 시스템

### 목적
- 주 4일 근무제 운영 원칙 및 규정을 명확히 문서화
- 휴무일 및 반차 신청·관리 기능 제공
- 휴무일 순환 및 승인 프로세스 자동화
- 팀 평균 근무 인원 관리 및 시각화

### 대상 사용자
- **일반 직원**: 휴무일 확인, 반차 신청, 본인 일정 확인
- **팀장(최고참)**: 반차 승인, 팀 일정 관리, 평균 근무 인원 확인
- **관리자**: 전체 운영 규칙 설정, 예외 처리, 통계 관리

## 📱 페이지 구성

### 1. 메인 페이지 (대시보드)
**URL**: `/` 또는 `/dashboard`

**구성 요소**:
- 환영 메시지
- 이번 주 일정 요약
- 빠른 액션 버튼 (반차 신청, 휴무일 조회)
- 최근 승인 내역 (팀장용)
- 평균 근무 인원 그래프 (팀장용)

### 2. 운영 원칙 페이지
**URL**: `/rules`

**구성 요소**:
- 계층적 섹션 구조
- 아코디언 또는 탭 메뉴
- 검색 기능 (선택사항)
- 인쇄 가능한 버전

### 3. 내 일정 페이지
**URL**: `/my-schedule`

**구성 요소**:
- 월별 캘린더 뷰
- 리스트 뷰 (선택)
- 휴무일 표시
- 반차 사용 내역
- 다음 휴무일 안내

### 4. 반차 신청 페이지
**URL**: `/half-day/request`

**구성 요소**:
- 날짜 선택기
- AM/PM 선택
- 사유 입력
- 사용 가능 여부 체크
- 주 단위 사용 제한 표시

### 5. 승인 관리 페이지 (팀장 전용)
**URL**: `/approval/pending`

**구성 요소**:
- 승인 대기 목록 (테이블)
- 승인/반려 버튼
- 승인 내역 (완료된 것)
- 필터링 (날짜, 직원, 상태)

### 6. 팀 일정 페이지 (팀장 전용)
**URL**: `/team/schedule`

**구성 요소**:
- 팀 전체 캘린더
- 직원별 색상 구분
- 평균 근무 인원 그래프
- 업무 공백 경고

### 7. 관리자 설정 페이지 (관리자 전용)
**URL**: `/admin/settings`

**구성 요소**:
- 직원 추가/수정/삭제
- 운영 규칙 설정
- 예외 처리 등록
- 시스템 통계

## 🛠 기술 스택 제안

### 프론트엔드
- **HTML/CSS/JavaScript**: 기본 웹 기술
- **캘린더 라이브러리**: FullCalendar.js 또는 Day.js + 직접 구현
- **UI 프레임워크**: Tailwind CSS 또는 Bootstrap (선택)
- **차트 라이브러리**: Chart.js (통계용)

### 백엔드
- **서버 언어**: PHP (기존 프로젝트와 일관성)
- **데이터베이스**: MySQL 또는 SQLite
- **세션 관리**: PHP Session

### 인증
- **로그인 시스템**: 이메일/비밀번호 또는 SSO (회사 시스템 연동)
- **세션 기반 인증**

### 배포
- **웹 서버**: Apache/Nginx
- **PHP 버전**: 7.4 이상

## 📊 데이터베이스 설계 (초안)

### users (직원 테이블)
```
- id (PK)
- email (unique)
- name
- password_hash
- role (employee/manager/admin)
- hire_date (입사일)
- probation_end_date (수습 종료일)
- current_off_day (현재 휴무 요일)
- created_at
- updated_at
```

### off_days (휴무일 테이블)
```
- id (PK)
- user_id (FK)
- date
- day_of_week
- shift_month (시프트 적용 월)
- created_at
```

### half_day_requests (반차 신청 테이블)
```
- id (PK)
- user_id (FK)
- date
- time_type (AM/PM)
- reason
- status (pending/approved/rejected)
- approved_by (FK to users)
- approved_at
- created_at
```

### approval_logs (승인 로그 테이블)
```
- id (PK)
- request_id (FK)
- approver_id (FK)
- action (approve/reject)
- comment
- created_at
```

### exceptions (예외 처리 테이블)
```
- id (PK)
- user_id (FK)
- exception_type
- description
- approved_by (FK)
- date_from
- date_to
- created_at
```
